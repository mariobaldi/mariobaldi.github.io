OSPF

I router sono stati configurati secondo la divisione in aree mostrata in figura.

Sono disponibili una serie di informazioni riguardanti i vari router:

  • la configurazione: Cisco 2503a, 2503b, 2503c, BayNetworks AN1 e AN2;
  • la tabella di routing: Cisco 2503a, 2503b, BayNetworks AN1 e AN2;
  • la base dati dei link state advertisement: Cisco 2503b, BayNetworks AN1.
Abbiamo realizzato alcune catture di pacchetti sulla rete locale tra i Cisco 2503b e 2503c con un analizzatore di protocollo Sniffer della Network General:
  • scambio di messaggi hello;
  • scambio di messaggi link state update e link state acknowledgment;
  • sincronizzazione delle basi dati mediante messaggi database description;
  • ulteriori scambi di messaggi: prima, seconda e terza cattura.
Abbiamo inoltre realizzato altre configurazioni del dominio di routing OSPF.
  • Aggregazione delle destinazioni della backbone area e dell'area 2:
    • base di dati dei link state advertisement di AN1,
    • base di dati dei link state advertisement di 2503b,
    • dettaglio dei link state advertisement di tipo 3 inviati nella backbone area per annunciare le destinazioni dell'area 2.
  • Dichiarazione dell'area 1 come stub area:
    • impostazione e visualizzazione della configurazione di 2503a,
    • visualizzazione della base di dati dei link state advertisement di 2503a.
  • Inibizione dell'invio di link state advertisement di tipo 3 (summary LSA) nell'area 1 dichiarata stub area:
    • impostazione e visualizzazione della configurazione di 2503a,
    • visualizzazione della base di dati dei link state advertisement di 2503a.
  • Creazione di un virtual link attraverso l'area 2 per rendere AN1 area border router e definire l'area 3.
